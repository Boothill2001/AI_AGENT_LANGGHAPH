1 AI Agent là gì? (giải thích từ đầu)

Chatbot thường chỉ trả lời — nhận câu hỏi, sinh text, xong. Agent thì hành động: nó có bộ tools và tự quyết định gọi tool nào, theo thứ tự nào, để hoàn thành yêu cầu phức tạp.

Ví dụ yêu cầu: "Tra cứu RRF là gì, tính 1/(60+1) + 1/(60+3), rồi lưu báo cáo." Chatbot chịu chết. Agent thì:

1️⃣ Plan: "cần 3 bước — tra cứu → tính toán → lưu file"
2️⃣ Gọi knowledge_search → nhận kết quả
3️⃣ Gọi calculator → nhận kết quả
4️⃣ Gọi save_report → dừng chờ con người duyệt (ghi file = nhạy cảm)
5️⃣ Tổng hợp câu trả lời cuối

💡 Vòng lặp "suy nghĩ → gọi tool → nhận kết quả → suy nghĩ tiếp" chính là planner/executor loop — trái tim của mọi agent.

2 Ba công nghệ chính

🕸️

LangGraph

Dựng agent dạng đồ thị trạng thái: mỗi node là 1 bước (planner, tools), cạnh quyết định đi đâu tiếp. Kiểm soát được, checkpoint được, pause/resume được — hơn hẳn vòng while tự viết.

🔌

MCP — Model Context Protocol

Chuẩn mở kết nối LLM ↔ tools qua giao thức thống nhất — như "USB-C cho AI". Tool viết 1 lần trong MCP server, agent nào (kể cả Claude Desktop) cũng cắm vào dùng được.

✋

Human-in-the-loop

Hành động nhạy cảm (ghi file, gửi mail, thanh toán) phải được người duyệt. LangGraph interrupt() làm đồ thị đứng lại giữa chừng — model không thể bypass.

🛡️

Guardrails

Cap số vòng lặp (chống chạy vô hạn), catch lỗi tool (không crash), calculator dùng AST whitelist thay vì eval(), graceful khi RAG API offline.

  1. Chỉ vào sidebar tools: "Tools không hard-code trong agent — chúng được serve qua MCP server thật, agent load qua MCP client. Đây là chuẩn công nghiệp nối LLM với internal APIs."

  5. Chốt hạ hệ sinh thái: "knowledge_search đang gọi vào chính RAG API của project kia — 2 repo nối thành 1 hệ thống: RAG = knowledge layer, agent = orchestration layer." 💥 Diễn sâu: tắt RAG API, hỏi lại — agent báo offline nhưng vẫn hoàn thành phần tính toán.

8 Câu hỏi phỏng vấn thường gặp (bấm để mở)

LangGraph khác gì LangChain?

LangChain là bộ toolkit (models, tools, prompts). LangGraph xây trên đó để dựng agent có trạng thái dạng đồ thị: node/edge tường minh, checkpoint, pause/resume (interrupt), replay. Vòng while tự viết không có mấy thứ đó.

ReAct và planner/executor khác gì?

ReAct đan xen "Thought → Action → Observation" từng bước — linh hoạt nhưng dễ lạc đề với task dài. Planner/executor tách vai: lập kế hoạch rồi thực thi có kiểm soát. Project này là planner/executor dạng vòng lặp: mỗi kết quả tool quay về planner để re-plan.

MCP là gì, sao không viết tool thẳng vào agent?

MCP là chuẩn mở kết nối LLM ↔ tools (stdio/HTTP). Tool viết thẳng vào agent thì chết dính với agent đó; expose qua MCP server thì viết 1 lần, mọi client dùng được (kể cả Claude Desktop), schema chuẩn, quản lý permission tập trung.

Làm sao chặn agent chạy vô hạn / phá hoại?

Nhiều lớp: (1) cap số vòng lặp — quá hạn ép trả lời ngay; (2) human-in-the-loop cho hành động nhạy cảm; (3) tool lỗi được catch trả message thay vì crash; (4) calculator dùng AST whitelist chứ không eval(); (5) recursion_limit của LangGraph là chốt chặn cuối.

Multi-agent thật thì khác gì demo này?

Cùng nguyên lý nhưng nhiều planner chuyên biệt (researcher, writer, reviewer...) giao tiếp qua state chung hoặc message passing; thêm supervisor/routing pattern. LangGraph hỗ trợ subgraph — kiến trúc demo mở rộng thẳng lên được.

Đưa lên production cần thêm gì?

Observability (trace từng tool call — LangSmith/OpenTelemetry), eval suite cho agent behavior, timeout + retry cho tool, auth cho MCP server, checkpoint bền (Postgres thay MemorySaver), rate limiting và cost tracking.
